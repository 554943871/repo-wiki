# Page Page Guidance

Page 页面解释用户可见页面、入口、导航、页面状态、可见区域、页面级交互，以及页面如何承接 flow、module 和 model。

## 应该帮助读者回答

- 这个页面服务哪个用户目标或业务场景。
- 用户从哪里进入这个页面。
- 页面可以跳转到哪里，什么时候返回。
- 页面有哪些关键可见状态、空态、异常态或禁用态。
- 页面有哪些稳定可见区域、主要组件、弹窗、抽屉、tab、占位区或操作区。
- 哪些业务条件会导致页面形态、组件编排、可操作项或出口路由变化。
- 页面级交互什么时候触发接口、能力或数据读取，用户能看到什么结果。
- 页面依赖哪些 flow、module 和 model。
- 哪些路由、接口、组件或代码位置能帮助验证页面行为。

## 适合写

- 页面目标和入口。
- 页面导航和返回路径。
- 用户可见状态和关键展示条件。
- 页面区域、稳定组件、弹窗、抽屉、tab、占位区和页面级 layout；空间关系重要时使用 Page Layout block。
- 页面多态：不同业务条件下的页面形态、组件编排、可见内容、可操作项和出口差异。
- 状态矩阵：业务状态、接口状态或 UI 展示态如何投射成用户可见结果、空态、异常态、禁用态和可操作项。
- 出口路由：用户可见动作、触发条件、目标页面、必要参数和返回路径。
- 页面级后端交互：触发时机、接口 / 能力、入参来源、页面影响和证据；只解释用户可见影响，不展开后端调用链。
- 页面级关键交互：跨组件、跨页面或多步用户旅程；必要时用 sequence 补充。
- 相关 flows、modules 和 models。
- 理解页面行为所需的关键路由、接口或数据来源；当页面有多个入口、route、action、event 或 exposed capability 时，使用 Public Surface table 说明 stable interaction points、使用者、稳定行为和 owner boundary。
- 仓库相对代码锚点。

## 避免写

- 完整 DOM 树。
- CSS、样式 token 或组件库细节。
- 组件清单式盘点。
- 后端调用链。
- 每个字段的展示规则全集。
- 像素级视觉稿还原、截图归档或样式 token 说明。
- 组件内部 layout、组件多态、组件内部事件链；这些只有在读者理解页面责任必须知道时才概括，详细内容应下钻到组件说明或相关 module。
- 把页面状态矩阵写成后端状态枚举全集，或把 route 参数全集写成导航图。
- 把页面当成 module 的能力边界替代品。
- 把 public surface 写成组件树、private handlers、文件树或完整接口字段索引；这些只作为 secondary code evidence。

## 推荐表达

Page 页面可以自然组合：

- 页面定位。
- 入口（Entry points）。
- 对外入口（Public Surface table）：当入口或暴露能力需要横向比较时。
- 页面导航（Navigation）。
- 页面布局（Page Layout block）：当区域、组件编排、弹窗 / 抽屉 / tab 或页面多态影响读者理解时。
- 可见状态（Visible states）。
- 状态矩阵（State matrix）：当状态、条件、可见结果和可操作项需要横向对照时。
- 出口路由（Exit routes）。
- 页面级后端交互（Page-level backend interactions）。
- 关键交互（Key interactions）。
- 相关流程（Related flows）。
- 相关模块（Related modules）。
- 相关模型（Related models）。
- 代码锚点（Code anchors）。

这些是写作建议，不是固定字段。

## LLM 语义检查问题

- 页面是否从用户可见行为出发，而不是从组件树或路由表出发？
- 入口和跳转是否能被代码、路由、产品材料或用户确认支撑？
- 页面级 public surfaces 是否说明 stable interaction points、使用者、稳定行为和 owner boundary，而不是列举组件或后端调用链？
- 页面 layout 是否只表达稳定可见区域、页面级组件编排、弹窗 / 抽屉 / tab 和必要下钻，而不是 DOM/CSS/组件库实现？
- 页面多态是否说明了触发条件、页面形态、组件编排、可操作项、出口和证据？
- 页面状态是否区分了业务状态、接口状态和 UI 展示态，并投射到用户可见结果、空态、异常态、禁用态和可操作项？
- 页面级后端交互是否只说明触发时机、接口 / 能力、入参来源、页面影响和证据，没有展开后端调用链？
- 页面级关键交互是否停留在页面级或跨组件旅程，组件内部细节是否下钻而不是塞回页面页？
- 是否链接到相关 flow、module 和 model，而不是重复它们的完整内容？
- 是否避免把后端调用链或模块职责写进页面页？
