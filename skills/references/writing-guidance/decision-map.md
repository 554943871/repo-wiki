# Decision Map Guidance

`wiki/06-decisions.md` 记录当前仍影响系统理解的重要决策和取舍。它不是历史讨论清单。

## 应该帮助读者回答

- 当前系统有哪些关键约束。
- 哪些取舍会影响模块、流程或模型理解。
- 哪些事情是团队明确不做的。
- 哪些决定未来容易被误解或误改。

## 适合写

- 当前仍有效的架构或产品约束。
- 有真实替代方案的设计取舍。
- 重要的 no-go 或 deliberate non-goal。
- 影响多个模块、流程或模型的决定。
- 需要给未来维护者解释的非显然选择。

## 避免写

- 纯历史讨论流水。
- 历史文件目录。
- 没有取舍的普通事实。
- 只影响一个局部代码分支的实现说明。
- 已经失效但没有当前影响的决定。

## 推荐表达

每条 decision 用短块表达：

```md
## {决策标题}

当前决策：...

为什么重要：...

排除的做法：...

相关 wiki 页面：
- ...
```

## LLM 语义检查问题

- 这条内容是否仍影响当前系统？
- 是否能看出做了什么取舍，而不只是描述事实？
- 是否说明了为什么不是另一种做法？
- 是否避免把 `06-decisions.md` 变成历史索引？
