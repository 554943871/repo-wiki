# Page Navigation Block

用于解释页面之间的用户可见入口、跳转关系和返回路径。

## 适合使用时机

- 一个页面有多个入口或出口。
- 用户路径需要跨多个页面理解。
- 页面跳转容易被误写成后端调用链或内部组件交互。

## 推荐表达

优先使用 Mermaid `flowchart` 或简洁导航表。

```md
| From page | User-visible action | To page | Return or next step | Evidence |
| --- | --- | --- | --- | --- |
| ... | ... | ... | ... | ... |
```

## 写作要求

- 每条边说明用户可见动作或入口。
- 区分页面跳转、页面内部状态变化和后端调用。
- 路由参数只在影响读者理解目标页面时写。
- 入口、路由和目标页面要有代码、路由配置、产品材料或用户确认支撑。

## 避免

- 把页面跳转图画成业务流程主活动图。
- 把后端接口调用画成页面跳转。
- 把组件内部交互提升成页面跳转。
- 把完整路由参数、接口字段或展示条件都塞进边标签。
