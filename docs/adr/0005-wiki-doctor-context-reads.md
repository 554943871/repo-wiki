# Wiki Doctor Context Reads

`wiki-doctor` may read locally available target-repository context outside `wiki/` while refreshing existing wiki pages. This includes source code, tests, routes, config, root README, docs, ADRs, AGENTS files, and other files that help understand names already present in the wiki, preserve links and evidence anchors, avoid meaning loss during a reader-facing rewrite, or write direct evidence-grounded current repo facts into stable wiki pages.

This makes `wiki-doctor` responsible for scoped evidence-grounded stable updates during a refresh, but not for queue-style drift classification. Outside-wiki files are allowed fact sources when the evidence is direct, local, readable, and cited. If a safe update depends on conflicting evidence, broad code/wiki comparison, missing coverage classification, or a fact decision that is not directly supported by inspected evidence, the item is `drift_or_coverage_suspect` and should go through `wiki-drift-radar` or focused user confirmation.

Consequences: `wiki-doctor` no longer fails just because it needs to inspect source or supporting docs during a refresh, and it no longer has to route every source/wiki mismatch through `wiki-drift-radar`. Its write boundary remains the target repository's top-level `wiki/`, including allowed Whitebox artifacts. Whitebox source models and rendered views may be created from facts already present in the wiki or direct current repo evidence; source reads can authorize new Whitebox facts when those facts are clear, cited, and not merely incidental private implementation details.
