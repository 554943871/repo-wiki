# C2 Root Module Owner Pages

The wiki no longer treats Module Overview as a separate page family. Module content generation starts from confirmed C2 runtime units in `wiki/01-system.md`; each confirmed C2 runtime unit should have a C2 root module owner page under `wiki/04-modules/`. `wiki/04-modules/README.md` stays a flat Canonical Module Index, while hierarchy, cross-module topology, and drill-down routes are expressed inside module owner pages.

Consequences: former `module-map.md` pages must either become C2 root module owner pages or be reported as `meaning_loss_risk` until their C2 root boundary is clarified. Stable subsystem, code-module, or other lower-level module owner pages are optional drill-down pages: they may be created only when tied to a C2 root module through Whitebox internal parts, existing wiki content, repo evidence, or explicit user confirmation. Upper-level module pages may show lower-level canonical modules as internal parts, but they only summarize those internal modules and link to their owner pages; the lower-level owner pages keep their full responsibilities, ports, contracts, internals, and evidence. Supporting participants may appear in Whitebox diagrams and node notes, but their appearance does not promote them into canonical modules.
